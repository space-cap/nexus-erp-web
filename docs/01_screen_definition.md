# ERP 웹 UI 화면 목록 정의서

## 1. 문서 목적

이 문서는 현재 제작된 ERP Web Mock UI의 화면 구성을 정리한 화면 목록 정의서이다. 각 화면의 목적, 주요 사용자, 검색 조건, 목록 컬럼, 상세 정보, 주요 액션을 정의해서 이후 실제 웹 ERP 개발, DB 설계, API 설계, 화면 검토 회의에 사용할 수 있도록 한다.

현재 기준 화면은 다음 9개이다.

- 대시보드
- 거래처 관리
- 품목 관리
- 재고 현황
- 수주 관리
- 발주 관리
- 생산 계획
- 처리 이력
- 사용자 관리

## 2. 공통 화면 구조

모든 업무 화면은 같은 기본 구조를 사용한다.

| 영역 | 설명 |
| --- | --- |
| 좌측 메뉴 | 업무 메뉴 이동 영역 |
| 상단 헤더 | 전역 검색, 알림, 사용자 정보 |
| 화면 제목 영역 | 현재 업무명, 상위 업무 구분, CSV, 가져오기, 신규 버튼 |
| 요약 영역 | 조회 건수, 금액/수량 합계, 정상/진행 건수, 주의 상태 건수 |
| 필터 영역 | 검색어, 상태, 담당/구분, 기간 조건 |
| 목록 영역 | 업무 데이터 테이블 |
| 상세 패널 | 선택한 행의 상세 정보 |

## 5. 화면별 정의

## 5.1 대시보드

### 화면 목적

ERP의 주요 업무 상태를 한 화면에서 빠르게 확인하는 화면이다. 경영진, 관리자, 각 부서 책임자가 매출, 재고, 생산, 승인 대기 상태를 확인하는 용도로 사용한다.

### 주요 사용자

- 경영진
- ERP 관리자
- 영업/구매/생산/재고 팀장

### 표시 정보

| 구분 | 항목 |
| --- | --- |
| KPI | 수주 금액, 재고 경고, 생산 진행, 승인 대기 |
| 차트 | 월간 매출 흐름 |
| 처리 필요 | 납기 지연, 발주 승인, 재고 부족 |
| 재고 경고 | 안전재고 이하 품목 |
| 생산 일정 | 라인별 생산 진행률 |

### 주요 액션

| 액션 | 설명 |
| --- | --- |
| 새로고침 | 대시보드 데이터를 다시 조회 |
| 신규 | mock 신규 등록 모달 표시 |

### 향후 개발 고려사항

- 실제 ERP에서는 사용자 권한에 따라 KPI가 달라져야 한다.
- 경영진 화면과 실무자 화면을 분리할 수 있다.
- 지연, 부족, 승인 대기 항목은 상세 화면으로 바로 이동할 수 있어야 한다.

- 사업자번호 중복 체크가 필요하다.
- 회계 시스템과 연결될 경우 세금계산서 정보가 필요하다.
- 거래처별 담당자 연락처를 여러 명 등록할 수 있어야 한다.
